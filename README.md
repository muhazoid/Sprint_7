Тестирование API приложения Яндекс.СамокатДанный проект содержит набор автоматизированных тестов для проверки бэкенда учебного сервиса Яндекс.Самокат. Тесты охватывают основные сценарии: создание курьера, авторизацию, создание заказов и получение списка заказов.

# Вспомогательные методы (helpers/courier_helper.py)

## register_new_courier_and_return_login_password
Генерирует случайные логин, пароль и имя, отправляет запрос на создание курьера.Возвращает список [login, password, first_name], если регистрация прошла успешно (код 201), иначе пустой список.

## delete_courier
Удаляет курьера по его ID. Используется для очистки данных после тестов.

## get_courier_id
Выполняет авторизацию курьера и возвращает его ID из ответа сервера.

## generate_random_string
Генерирует строку заданной длины из букв нижнего регистра. Используется для создания уникальных тестовых данных.

# Тестовые данные (data.py)
## Urls
- BASE_URL — базовый URL тестируемого сервиса (https://qa-scooter.praktikum-services.ru).

## CourierMessages
Содержит константы с текстами ответов сервера:

- ERROR_MISSING_FIELD — ошибка при отсутствии обязательных полей.
- ERROR_DUPLICATE_LOGIN — ошибка при создании курьера с существующим логином.
- ERROR_AUTH_FAILED — ошибка авторизации (неверные данные).
- ERROR_MISSING_AUTH_FIELDS — ошибка при отсутствии логина или пароля.
- SUCCESS_CREATE — успешный ответ при создании курьера.
## OrderData
Содержит шаблоны данных для создания заказа:

- COLORS — словарь с вариантами выбора цвета самоката (BLACK, GREY, BOTH, NONE).
- BASE_ORDER — основной набор валидных данных для заказа.
- ALTERNATIVE_ORDER — альтернативный набор данных для параметризации тестов.

# Фикстуры (conftest.py)
## create_and_delete_courier
Фикстура для тестов, требующих наличия созданного курьера.

- Создает нового курьера через API перед тестом (setup).
- Возвращает словарь с данными курьера в тест.
- Удаляет курьера после завершения теста (teardown).

## create_and_delete_courier_with_response
Фикстура для тестов, проверяющих процесс создания курьера.

- Генерирует данные и отправляет запрос на создание курьера самостоятельно (не использует хелпер register_new_courier_and_return_login_password).
- Возвращает словарь, содержащий объект response, а также логин, пароль и имя.
- Позволяет проверить статус-код и тело ответа при создании.
- Удаляет созданного курьера после завершения теста.

# Тесты (tests/)
## test_create_courier.py
Проверяет ручку создания курьера:

- Успешное создание нового курьера.
- Создание курьера с уже используемым логином.
- Создание дубликата курьера.
- Создание курьера без обязательного поля.
- Создание курьера с пустым обязательным полем.

## test_login_courier.py
Проверяет ручку авторизации курьера:

- Успешная авторизация (возвращается id).
- Авторизация с неверным паролем.
- Авторизация с несуществующим логином.
- Авторизация без обязательного поля.
- Авторизация с пустым полем.

## test_create_order.py
Проверяет ручку создания заказа:

- Создание заказа с разными наборами данных (параметризация).
- Создание заказа без указания цвета.
- Создание заказа с цветом BLACK, GREY и обоими цветами.
- Попытка создания заказа с пустым телом запроса.

## test_get_orders.py
Проверяет ручку получения списка заказов:

- Возврат списка заказов без параметров (код 200).
- Наличие поля "orders" в ответе и проверка, что это список.
